# ESP32-S3 语音助手项目问题总结

## 概述
本项目在ESP32-S3平台上开发语音助手，遇到两个主要问题：
1. **内存不足问题**：内部SRAM（约512KB）不足，导致HTTPS请求失败
2. **栈溢出问题**：SSL握手过程中loopTask栈空间不足，导致系统崩溃

## 问题一：内存不足（内部SRAM耗尽）

### 症状
- HTTPS请求失败，内部堆剩余低于35KB阈值
- 语音识别成功后，对话和语音合成服务因内存不足无法执行
- 系统日志显示：`[NetworkManager] Internal heap insufficient: 30000 available, 35000 required`

### 根本原因
1. **音频缓冲区占用过大**：`MainApplication::audioBuffer`静态分配160KB（160,000字节）
2. **SSL客户端内存需求**：每个`WiFiClientSecure`实例需要30-60KB内部SRAM
3. **任务栈分配过大**：音频录音任务栈16KB，HTTP客户端池3个，SSL客户端池2个
4. **PSRAM未充分利用**：虽然ESP32-S3有8MB PSRAM，但关键缓冲区未使用PSRAM

### 已实施的解决方案

#### 1. 音频缓冲区PSRAM迁移
- **文件**: [src/MainApplication.h](src/MainApplication.h), [src/MainApplication.cpp](src/MainApplication.cpp)
- **更改**:
  - 将`audioBuffer`从静态数组改为动态指针：`uint8_t* audioBuffer;`
  - 添加`audioBufferInPSRAM`标志跟踪分配位置
  - 构造函数中优先尝试PSRAM分配，失败时回退到内部SRAM
  - 析构函数中添加内存释放

#### 2. 任务栈大小优化
- **文件**: [src/drivers/AudioDriver.cpp](src/drivers/AudioDriver.cpp)
- **更改**: 录音任务栈从16KB减少到12KB（节省4KB）

#### 3. 客户端池大小优化
- **文件**: [src/modules/NetworkManager.cpp](src/modules/NetworkManager.cpp)
- **更改**: HTTP客户端最大数量从3个减少到2个

- **文件**: [src/modules/SSLClientManager.cpp](src/modules/SSLClientManager.cpp)
- **更改**: SSL客户端最大数量从2个减少到1个

#### 4. 增强内存监控
- **文件**: [src/utils/MemoryUtils.h](src/utils/MemoryUtils.h), [src/utils/MemoryUtils.cpp](src/utils/MemoryUtils.cpp)
- **新增功能**:
  - `printDetailedMemoryStatus()`: 详细内存状态报告
  - `verifyPSRAMAllocation()`: PSRAM实际分配验证
  - `printPSRAMStatus()`: PSRAM详细状态
  - 定期内存监控任务（每30秒）

#### 5. PSRAM配置验证
- **文件**: [platformio.ini](platformio.ini)
- **确保配置**:
  ```ini
  -D BOARD_HAS_PSRAM
  -mfix-esp32-psram-cache-issue
  -D CONFIG_SPIRAM_USE_CAPS_ALLOC=1
  -D CONFIG_SPIRAM_USE_MALLOC=1
  ```

### 内存优化效果
- **释放160KB内部SRAM**：音频缓冲区移到PSRAM
- **减少约50KB堆占用**：客户端池和任务栈优化
- **内部SRAM空闲目标**：从30KB提升到50KB以上

## 问题二：栈溢出（SSL握手时loopTask栈空间不足）

### 症状
- 系统在语音合成阶段崩溃
- 错误信息：`Guru Meditation Error: Core 1 panic'ed (Unhandled debug exception)`
- 具体原因：`Stack canary watchpoint triggered (loopTask)`
- 发生时机：向火山语音合成API发起HTTPS请求时

### 错误回溯（关键部分）
```
/home/runner/work/esp32-arduino-lib-builder/esp32-arduino-lib-builder/esp-idf/components/mbedtls/mbedtls/library/sha512.c:449
/home/runner/work/esp32-arduino-lib-builder/esp32-arduino-lib-builder/esp-idf/components/mbedtls/mbedtls/library/entropy.c:174
...
C:/Users/Admin/.platformio/packages/framework-arduinoespressif32/libraries/WiFiClientSecure/src/ssl_client.cpp:271
C:/Users/Admin/.platformio/packages/framework-arduinoespressif32/libraries/HTTPClient/src/HTTPClient.cpp:1162
...
C:\Users\Admin\Documents\PlatformIO\Projects\test1/src/MainApplication.cpp:128
C:/Users/Admin/.platformio/packages/framework-arduinoespressif32/cores/esp32/main.cpp:50
```

### 调用链深度分析
```
main.cpp:loop() → MainApplication::update() → VolcanoSpeechService → 
NetworkManager → HTTPClient → WiFiClientSecure SSL握手 → mbedtls SHA512计算
```

### 根本原因
1. **loopTask默认栈大小不足**：Arduino框架默认可能只有8192字节
2. **SSL握手栈密集型**：mbedtls库的SHA512计算需要大量栈空间
3. **调用链过深**：多层函数调用累积栈使用
4. **mbedtls配置未优化**：启用不必要的特性和算法

### 已实施的解决方案

#### 1. 增加loopTask栈大小
- **文件**: [sdkconfig.defaults](sdkconfig.defaults)
- **配置**: `CONFIG_ARDUINO_LOOP_STACK_SIZE=24576`（24KB）

#### 2. 深度优化mbedtls SSL配置
- **文件**: [sdkconfig.defaults](sdkconfig.defaults)
- **关键优化**:
  ```ini
  # 只启用TLS1.2，禁用旧协议
  CONFIG_MBEDTLS_SSL_PROTO_TLS1_2=y
  CONFIG_MBEDTLS_SSL_PROTO_TLS1_1=n
  CONFIG_MBEDTLS_SSL_PROTO_TLS1=n
  
  # 只保留一个椭圆曲线（最小化栈使用）
  CONFIG_MBEDTLS_ECP_DP_SECP256R1_ENABLED=y
  # 禁用其他所有椭圆曲线
  
  # 只使用一个密码套件
  CONFIG_MBEDTLS_SSL_CIPHERSUITES="TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256"
  
  # 禁用SHA512以减少栈使用
  CONFIG_MBEDTLS_SHA512_C=n
  CONFIG_MBEDTLS_SHA384_C=n
  
  # 减少SSL缓冲区大小
  CONFIG_MBEDTLS_SSL_IN_CONTENT_LEN=512
  CONFIG_MBEDTLS_SSL_OUT_CONTENT_LEN=512
  CONFIG_MBEDTLS_SSL_MAX_CONTENT_LEN=512
  
  # 启用硬件加速
  CONFIG_MBEDTLS_HARDWARE_AES=y
  CONFIG_MBEDTLS_HARDWARE_MPI=y
  CONFIG_MBEDTLS_HARDWARE_SHA=y
  ```

#### 3. 更新platformio.ini引用配置
- **文件**: [platformio.ini](platformio.ini)
- **更改**:
  ```ini
  board_build.sdkconfig = sdkconfig.defaults
  -DCONFIG_MBEDTLS_SSL_MAX_CONTENT_LEN=512
  ```

### 其他任务栈配置检查
- **AudioDriver recordTask**: 12288字节（12KB） - 音频处理需要
- **AudioDriver playTask**: 4096字节（4KB） - 合理
- **MemoryMonitor任务**: 2048字节（2KB） - 合理

## 配置更改总结

### platformio.ini 关键更改
```ini
build_flags =
    ...
    -D BOARD_HAS_PSRAM
    -mfix-esp32-psram-cache-issue
    -D CONFIG_SPIRAM_USE_CAPS_ALLOC=1
    -D CONFIG_SPIRAM_USE_MALLOC=1
    -DCONFIG_MBEDTLS_SSL_MAX_CONTENT_LEN=512
    -DCONFIG_MBEDTLS_TLS_MEMORY_DEBUG=0
    -DCONFIG_MBEDTLS_INTERNAL_MEM_SIZE=163840

board_build.sdkconfig = sdkconfig.defaults
```

### sdkconfig.defaults 完整配置
```ini
# mbedTLS配置优化以减少内存和栈使用
CONFIG_MBEDTLS_SSL_MAX_CONTENT_LEN=512
CONFIG_MBEDTLS_SSL_PROTO_TLS1_2=y
CONFIG_MBEDTLS_SSL_PROTO_TLS1_1=n
CONFIG_MBEDTLS_SSL_PROTO_TLS1=n

# 禁用不必要的SSL特性
CONFIG_MBEDTLS_SSL_ALPN=n
CONFIG_MBEDTLS_SSL_SESSION_TICKETS=n
CONFIG_MBEDTLS_SSL_RENEGOTIATION=n

# 只启用一个椭圆曲线
CONFIG_MBEDTLS_ECP_DP_SECP256R1_ENABLED=y
# 禁用其他所有椭圆曲线...

# 只使用一个密码套件
CONFIG_MBEDTLS_SSL_CIPHERSUITES="TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256"

# 禁用不必要的哈希算法
CONFIG_MBEDTLS_SHA512_C=n
CONFIG_MBEDTLS_SHA384_C=n

# 减少SSL缓冲区
CONFIG_MBEDTLS_SSL_IN_CONTENT_LEN=512
CONFIG_MBEDTLS_SSL_OUT_CONTENT_LEN=512

# 任务栈大小调整
CONFIG_ARDUINO_LOOP_STACK_SIZE=24576
```

## 测试结果与待解决问题

### 已解决的问题
1. ✅ 音频缓冲区成功迁移到PSRAM，释放160KB内部SRAM
2. ✅ 内存监控系统增强，提供详细诊断信息
3. ✅ PSRAM可用性验证增强，确保分配可靠性
4. ✅ 各组件内存占用优化

### 仍存在的问题
1. ❌ **栈溢出问题仍然存在**：即使增加loopTask栈到24KB并优化mbedtls配置，SSL握手仍导致栈溢出
2. ⚠️ **网络超时问题**：Coze API请求出现`read Timeout`错误
3. ⚠️ **错误恢复机制**：API失败后系统状态管理需要改进

3. **调用链优化**：减少函数调用深度，避免深度嵌套
4. **静态分配替代栈分配**：将部分栈变量改为静态或堆分配

#### 针对网络问题

2. **连接池优化**：改进SSL客户端复用机制
